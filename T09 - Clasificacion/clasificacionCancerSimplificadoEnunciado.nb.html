Dividir los datos en muestras para entrenamiento y test

Para dividir los datos en una muestra de entrenamiento y otra de test se usa el comando createDataPartition. En el c√≥digo siguiente p representa la proporci√≥n de datos en la muestra de entrenamiento. La partici√≥n se lleva a cabo para cada nivel de la variable y que aparece como primer argumento. El resultado es un vector con los √≠ndices de las filas seleccionadas para formar parte de la muestra de entrenamiento.

Un aspecto importante, sobre todo pensando en la reproducibilidad de nuestro trabajo, es que si iniciamos al generador de n√∫meros aleatorios con un valor determinado, la secuencia de n√∫meros pseudo-aleatorios se va a repetir y por lo tanto podemos reproducir exactamente una simulaci√≥n estoc√°stica.R utiliza la funci√≥n set.seed(numero entero) para inicializar el generador de n√∫meros aleatorios.

set.seed(100)  # Para reproducir los mismos resultados
IndicesEntrenamiento <- createDataPartition(y = dataCancer$y,
                                            p = 0.75,
                                            list = FALSE)
Entrenamiento <- dataCancer[IndicesEntrenamiento,]
Test <- dataCancer[-IndicesEntrenamiento,]

Es importante verificar que la distribuci√≥n de la variable respuesta es similar en el conjunto de entrenamiento y en el de test. Por defecto, la funci√≥n createDataPartition() garantiza una distribuci√≥n aproximada.

El comando train

Se puede usar este comando √∫nico para aplicar un gran n√∫mero de m√©todos de clasificaci√≥n determinando (en caso necesario) los valores √≥ptimos de sus par√°metros mediante validaci√≥n cruzada u otros m√©todos de remuestreo. Para usar train en general es necesario:

  • Elegir el m√©todo de clasificaci√≥n que queremos usar. El cat√°logo de todos los m√©todos disponibles se puede consultar en este enlace. Una informaci√≥n t√©cnica detallada de cada m√©todo se puede obtener con el comando getModelInfo.

  • Si el m√©todo de clasificaci√≥n requiere determinar par√°metros, es necesario fijar cu√°les y en qu√© rango de valores.

  • Tambi√©n hay que definir el m√©todo de remuestreo que se va a utilizar para determinar estos par√°metros.
